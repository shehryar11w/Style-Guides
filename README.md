# Style Guides

A comprehensive repository containing style guides, design patterns, and ready-to-use boilerplates that I use for multiple frameworks across different programming languages. This helps maintain a familiar structure for different projects on a framework and makes collaboration easier

## Overview

This repository serves as a centralized collection of:
- **Complete style guides** for each framework
- **Directory structure patterns** for organized code organization
- **Ready-to-use boilerplates** following best practices
- **Comprehensive documentation** covering all aspects of design patterns

## Frameworks Included

### Python
- [Django](./python/django/README.md) - Full-stack web framework
- [FastAPI](./python/fastapi/README.md) - Modern API framework with AI/ML focus

### JavaScript/TypeScript
- [React](./javascript/react/README.md) - UI library for building user interfaces
- [Next.js](./javascript/next/README.md) - React framework for production
- [React Native](./javascript/react-native/README.md) - Mobile app development framework
- [Node.js](./javascript/nodejs/README.md) - Server-side JavaScript runtime with Express.js

## How to Use

1. **Browse Documentation**: Navigate to the framework you're interested in and read the comprehensive `README.md` file
2. **Use Boilerplates**: Copy the `boilerplate/` folder contents to start a new project with best practices already implemented
3. **Follow Patterns**: Reference the documentation to maintain consistency across your projects

## Contributing

When adding new frameworks or patterns:
- Follow the established directory structure: `language/framework/`
- Include a comprehensive `README.md` with all sections outlined in the documentation structure
- Provide a complete, runnable boilerplate in the `boilerplate/` folder
- Ensure all code follows the documented style guidelines
